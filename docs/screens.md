## 1. ダッシュボード

### 上部：主要KPIカード

| カード | 内容 |
|--------|------|
| アクティブユーザー数 | 今月 / 先月比 |
| 離反予備軍数 | 条件カスタマイズ可 |
| 新規顧客数 | 今月 / 先月比 |
| 平均客単価 | 今月 / 先月比 |
| 平均利用パートナー数 | 今月 / 先月比 |
| 新規利用パートナー数 | 今月 / 先月比 |

### 中部：グラフ

| グラフ | 内容 |
|--------|------|
| ログイン推移 | 日別アクティブユーザー数の折れ線 |
| 注文推移 | 日別注文件数・金額の折れ線 |
| 離反率推移 | 月別の離反率トレンド |

### 下部：アクションリスト

| リスト | 内容 |
|--------|------|
| 要対応ユーザー | 離反リスク高・カゴ落ち等、今すぐアクションすべきユーザー |
| 直近の配信結果 | 最新の配信とその開封率・反応サマリ |

### 離反/新規分析

ダッシュボード上ではサマリ表示。クリックで詳細画面に遷移。

| 項目 | 仕様 |
|------|------|
| 比較期間 | 自由選択（先週vs今週、先月vs今月、任意期間） |
| 切り口 | 顧客別 → パートナー別 → 商品別（3階層） |
| 基準 | 注文回数・注文金額・注文頻度（選択 & 組み合わせ可） |

## 2. ユーザー一覧

### 表示項目

| 項目 | 内容 |
|------|------|
| 顧客コード | 既存の顧客コード |
| 顧客名 | 店舗名 |
| 住所 | エリア把握用 |
| 最終ログイン | 日時 + 経過日数 |
| 最終注文日 | 日時 + 経過日数 |
| 今月注文金額 | 合計額 |
| 今月注文回数 | 回数 |
| 利用パートナー数 | 何社利用中か |
| ステータス | アクティブ / 離反リスク / 離反 / 新規 |
| LINE連携 | 連携済み / 未連携 |

### フィルタ・ソート

| フィルタ | 例 |
|---------|-----|
| ステータス | アクティブ / 離反リスク / 離反 / 新規 |
| 最終ログイン | N日以内 / N日以上前 |
| 注文金額 | 範囲指定 |
| 利用パートナー | 特定パートナーの利用有無 |
| 期間 | 任意期間指定 |

### アクション
- クリックでユーザー詳細に遷移
- チェックボックスで複数選択 → 一括配信

## 3. ユーザー詳細

### ヘッダー：基本情報

| 項目 | 内容 |
|------|------|
| 顧客コード / 店舗名 / 住所 | 基本属性 |
| ステータス | アクティブ / 離反リスク / 離反 / 新規 |
| LINE連携状況 | 連携済み / 未連携 |
| 最終ログイン / 最終注文日 | 経過日数付き |

### タブ①：注文履歴
- 注文日 / 金額 / パートナー / 商品一覧
- 注文金額・頻度の推移グラフ
- 利用パートナーの変遷

### タブ②：行動履歴
- ログイン日時
- ページ閲覧履歴
- サイト内検索キーワード
- カゴ落ち履歴

### タブ③：コミュニケーション履歴
- 送信済み LINE / メール / SMS の一覧
- 開封・クリック状況
- Kintone の対応履歴（連携時）

### タブ④：アクション
- この顧客に LINE / メール / SMS を送信
- メモを追加（Kintone に保存）

## 6. 自動配信ルール設定

> **TODO**: 詳細設計未完了

### 概要
条件（トリガー）とアクション（配信内容）を設定し、条件に合致したユーザーに自動でコミュニケーションを送る画面。

### 想定ユースケース
- 最終ログインからN日経過 → LINEリマインド
- カゴ落ち発生 → 翌日メール送信
- 特定ページ閲覧 → フォローアップメール
- 注文頻度低下 → 離反防止LINE

### 想定機能
- トリガー条件の設定
- アクション（チャネル + メッセージ）の設定
- ルールの有効/無効切り替え
- ルール実行履歴
