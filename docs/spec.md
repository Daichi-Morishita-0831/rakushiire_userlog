# rakushiire-crm 仕様書

## 1. プロジェクト概要

### 目的
rakushiire.com のユーザー行動データを活用し、CRMチームが顧客の離反防止・アップセル・クロスセルを効率的に実行できる管理ツールを構築する。

### KPI
| KPI | 内容 |
|-----|------|
| 離反率低下 | 既存顧客の継続利用率向上 |
| アップセル | 客単価向上 |
| クロスセル | 利用パートナー数向上 |

### 利用者
- CRMチーム

### システム構成
- 新規管理画面（Webアプリケーション）
- rakushiire.com / vege-tal.com とはデータ連携（同一システムに組み込まない）
- モバイル対応: 不要

## 5. 離反/新規分析

### 離反分析
「以前は買っていたが、最近買わなくなった」顧客・パートナー・商品を検知する。

### 新規分析
「以前は買っていなかったが、新たに買い始めた」顧客・パートナー・商品を検知する。

### 共通仕様

| 項目 | 仕様 |
|------|------|
| 比較期間 | 自由選択（先週vs今週、先月vs今月、任意期間） |
| 切り口 | 顧客別 → パートナー別 → 商品別（3階層） |
| 基準 | 注文回数・注文金額・注文頻度（選択 & 組み合わせ可） |
| 表示 | ダッシュボードにサマリ → クリックで詳細画面に遷移 |

### 3階層の例

**離反**
| レベル | 例 |
|--------|-----|
| 顧客別 | 「A店が今月注文ゼロ」 |
| パートナー別 | 「A店は精肉B社を使わなくなった」 |
| 商品別 | 「A店は鶏ももを買わなくなった」 |

**新規**
| レベル | 例 |
|--------|-----|
| 顧客別 | 「今月新しくD店が注文開始した」 |
| パートナー別 | 「A店が今月から鮮魚C社を使い始めた」 |
| 商品別 | 「A店が今月からサーモンを買い始めた」 |

## 6. セグメント機能

### セグメントの種類

| 種類 | 仕組み | 用途 |
|------|--------|------|
| 動的セグメント | 条件を保存。毎日自動で対象者が更新される | 自動配信ルール向き |
| 静的セグメント | 作成時点のユーザーリストを固定 | キャンペーン等の1回限りの配信向き |

保存時に「動的 / 静的」を選択可能。

### 条件設定
- 属性条件（顧客コード、エリア、ステータス、利用パートナー、LINE連携）
- 行動条件（ログイン、ページ閲覧、サイト内検索、注文、カゴ落ち、流入元）
- AND / OR で複数条件を組み合わせ可能

---

## 7. 自動配信ルール

### トリガー条件
- 最終ログインからN日経過
- 注文頻度がN%低下 / N日間注文なし
- カートに入れたがN時間以内に未注文
- 特定ページを閲覧した
- 特定キーワードで検索した
- 動的セグメントに新しく入った / 出た

### アクション
- チャネル: LINE / メール / SMS（単一 or 複数選択可）
- メッセージ: テキスト入力 + 差し込み変数
- 送信タイミング: 即時 / N時間後 / N日後
- サイト内バナー表示（Phase 2）

### 重複防止
- 同じユーザーには最低N日間空ける（N日は設定可能）
- 自動配信の上限数は設けない

## 8. 効果測定

| 指標 | 内容 |
|------|------|
| 開封率 | メッセージを開いた割合 |
| クリック率 | リンクをクリックした割合 |
| 配信後N日以内の注文率 | 送った人のうち何%が注文したか |
| 配信後N日以内の注文金額 | 配信対象者の合計注文額 |
| 配信後のログイン率 | 送った人のうち何%がログインしたか |
| ブロック/配信停止率 | LINEブロック・メール配信停止の割合 |
| 未配信者との比較 | 送った人 vs 送らなかった人の注文率差 |

CSVエクスポート対応。

## 10. Phase計画

### Phase 1（優先）
- 全7画面の基本機能
- LINE / メール / SMS 配信（単一・複数チャネル）

- CSVエクスポート
- Kintone連携

- 顧客プロフィール強化
